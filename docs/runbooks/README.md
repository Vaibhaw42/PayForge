## Rules

- One runbook per operational scenario (deploy, rollback, incident triage, secret rotation, DB restore, etc.).
- Every runbook must be executable by a stranger — no tribal knowledge.
- Every runbook has: preconditions, steps, verification, rollback, escalation.
