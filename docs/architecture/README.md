## Rules

- Each phase gets its own file: `phase-N.md`.
- Every architecture doc must include:
  - Context (why this phase exists)
  - High-level design (Mermaid diagram)
  - Component responsibilities
  - Sequence diagram(s) for the key flows
  - Non-functional targets (latency, throughput, availability)
  - Failure modes
  - Trade-offs considered (link ADRs)
